# Identidad Visual Inicial

Esta guia sintetiza la direccion visual tomada de la antepropuesta compartida para `ATAR`.

## Objetivo

Construir una interfaz que comunique:

- tecnologia;
- confianza;
- operacion industrial real;
- claridad comercial;
- escalabilidad regional.

## Rasgos Visuales

- estilo moderno y sobrio;
- contraste alto entre fondos oscuros y superficies claras;
- acentos en azul electrico y violeta;
- composicion limpia con bloques amplios y tipografia fuerte;
- combinacion de lenguaje corporativo y enfoque industrial.

## Paleta Base

- `#2563EB`
- `#5B3DFF`
- `#0B1324`
- `#64748B`
- `#F4F6FA`

Uso sugerido:

- azul y violeta para CTAs, foco y acentos;
- azul oscuro para fondos premium y secciones institucionales;
- grises frios para soporte, texto secundario y superficies;
- blanco roto para secciones de contenido y lectura.

## Tipografia

La antepropuesta sugiere una familia del tipo `Inter`:

- titulos: peso semibold o bold;
- subtitulos: semibold;
- texto corrido: regular;
- espaciado amplio y lectura clara.

## Sistema Visual

La home debe comunicar:

1. propuesta de valor en segundos;
2. confianza y verificabilidad;
3. marketplace activo;
4. proceso simple de compra industrial;
5. capacidad tecnologica y escalable.

## Componentes Clave

- `hero` con CTA principal y CTA secundario;
- cards de beneficios y diferenciales;
- grilla de categorias;
- bloque de proveedores destacados;
- seccion `como funciona` por pasos;
- banda final de conversion con mensaje institucional.

## Lenguaje Fotografico

- imagenes industriales reales;
- plantas, bobinas, galpones, operadores y logistica;
- escenas profesionales, no genericas;
- foco en credibilidad, escala y operacion.

## Iconografia

- lineal;
- simple;
- limpia;
- de lectura rapida;
- coherente con un entorno B2B.

## Aplicacion Inicial En El Proyecto

Esta identidad ya se reflejo en:

- `apps/web/src/app/page.tsx`;
- `apps/web/src/app/layout.tsx`;
- `apps/web/src/app/globals.css`.

## Siguiente Evolucion Recomendada

- crear un set de componentes UI reutilizables;
- definir tokens de color y espaciado compartidos;
- incorporar assets reales del branding definitivo;
- extender la misma identidad a `mobile` y al panel autenticado.
